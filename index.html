Web Security

How would you advise a potential client to secure his/her website from web security attacks? Note: the word count for your answer should not be greater than 250 words

Here are Some of the best practices when it comes to Web Security that are widely accepted and proven to be efficient. Inplementing smart Sanitization practice of data, involves ensuring that the data a system accepts is in the right format. This can be done by using a well-known third-party solution. Like, NodeJS, the sanitize-html package for sanitizing HTML code and ORM(Object Related Mapping) library for sanitizing the SQL related code. When making use of third-party application for the website, ensure that the choice of third-party application is a well-known trusted, tested and the company has good track record of frequently use and download. It is smart to make use of Secure authorization and authentication practices when a user is accessing the your website by using an open-standard authorization protocol or framework like such as OneLogin and Okta. A network shares a lot of data across nodes (like servers). Data could be easily be compromised also via the hosting platform. Therefor it is important to have Network security in place for your website to protect the exchange of data across the network. All this security will help doing the following: ”Network security consists of the policies, processes and practices adopted to prevent, detect and monitor unauthorized access, misuse, modification, or denial of a computer network and network-accessible resources.” -Wikipedia Other sources: Moodle, Web and Communications Technology, web security
